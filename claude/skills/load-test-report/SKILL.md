# Load Test Report Skill

Analyze Artillery load test output and produce a structured performance summary.

## Steps

1. **Read `report.json`** from the project root (generated by `npm run artillery-load-test`)
2. **Read SLO targets** from `src/scripts/load-testing/artillery.yml` (phases, rampTo rates, http timeout)
3. **Extract key metrics** from the report:
   - `aggregate.latency`: p50, p95, p99, max (in ms)
   - `aggregate.rps`: mean requests/sec, min, max
   - `aggregate.errors`: total errors and error rate %
   - `aggregate.codes`: HTTP status code breakdown
   - `aggregate.scenarioDuration`: per-scenario latency stats
4. **Evaluate against SLOs**:
   - p95 latency < 500ms → PASS / FAIL
   - p99 latency < 1000ms → PASS / FAIL
   - Error rate < 1% → PASS / FAIL
   - HTTP timeout (from yml, default 10s) — flag any requests exceeding it
5. **Output the report** in this format:

```
## Load Test Report
Date: <timestamp from report>
Duration: <total duration>
Target: <target URL>

### Summary
| Metric         | Value     | SLO      | Status |
|----------------|-----------|----------|--------|
| p50 latency    | Xms       | —        | —      |
| p95 latency    | Xms       | <500ms   | PASS/FAIL |
| p99 latency    | Xms       | <1000ms  | PASS/FAIL |
| Error rate     | X%        | <1%      | PASS/FAIL |
| Mean RPS       | X req/s   | —        | —      |
| Max RPS        | X req/s   | —        | —      |

### HTTP Status Codes
<breakdown of 2xx/4xx/5xx>

### Errors
<list top errors if any, with counts>

### Scenarios
<per-phase latency breakdown if available>

### Verdict
PASS ✓ / FAIL ✗ — <1-line summary of what passed or what to fix>
```

6. **If `report.json` does not exist**, output: `No report.json found. Run: npm run artillery-load-test first.`
7. **If the report shows failures**, suggest next steps:
   - High p99: Check slow DB queries, add Redis caching, review @LogExecution for synchronous blocking
   - High error rate: Check for timeout configs in Fastify, SQS backpressure, MySQL connection pool exhaustion
   - Low RPS ceiling: Check `maxVusers` setting and Node.js event loop blocking
